32 *

34 * parses a defined table chunk size based on either item count or buffer size
35 *
36 * fast string to number conversion and splits up chunks for parallel processing
37 *
38 * supports bi-directional flow control between internal CPU process and IO
39 * native string struct is formatted to be byte-for-byte compatible with MySQL
40 *

42 * target formats for import and export include CSV, JSON, XML, KML, OBJ...
43 *
44 * data container and tree structure described separately
45 *
